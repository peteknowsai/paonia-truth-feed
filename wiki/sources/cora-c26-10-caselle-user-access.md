# CORA C 26-10 — Caselle ERP User Access Report

**Type:** CORA response package
**Request filed:** 2026-04-01 (Wed)
**Receipt acknowledged:** 2026-04-02 (Thu)
**Extension issued:** 2026-04-07 — 7-day extension invoked; extended deadline April 16, 2026
**Fulfilled:** 2026-04-16 — delivered on the extended deadline
**Fee:** not stated in available records

## Request

The request sought the Caselle ERP "User Access (Application Rights) Report" — the report listing which users hold which permissions across the Town's Caselle accounting modules.

## What Was Produced

The Town produced the Application Rights Report: a per-task listing of Caselle module rights with Allowed Users and Denied Users for each. The report is dated April 8, 2026.

## The As-Of Gap

The report the Town ran is dated April 8, 2026. It therefore reflects the state of permissions **as of April 8** — the day it was generated — not as of the April 1 filing date.

More to the point, the Application Rights Report is a snapshot. It shows who held what access on the day it was run. It carries **no historical change record**: no indication of who previously held a given permission, when any permission was added or removed, or who made the change. A reader of this report can see the current configuration but cannot reconstruct how that configuration was arrived at.

That gap is what prompted the follow-on request, [[cora-c26-17-caselle-permissions-audit-log|C 26-17]], which sought the permission-change communications and the Caselle audit log (the Management Tracking Report) covering the period of changes. See [[cora-c26-17-caselle-permissions-audit-log]] for what that request produced and what the Town declined to produce.

## Significance

The Application Rights Report is the relevant record for analyzing segregation of duties in the Town's financial system — who can enter, approve, and post transactions, and whether those rights overlap in a single account. It is a starting point for that analysis, not a complete one: it documents the present state but not the change history needed to establish when a control was weakened or removed. See [[who-controls-the-money]].
