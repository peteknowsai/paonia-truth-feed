# CORA Request: Caselle System Audit Logs

**Date:** 2026-04-12

**To:** Samira Vetter, Town Clerk, Town of Paonia (SamiraV@townofpaonia.com)

**Re:** Colorado Open Records Act request — Caselle financial system audit and activity logs

---

Dear Ms. Vetter,

Pursuant to the Colorado Open Records Act, C.R.S. 24-72-201 et seq., I request the following records from the Town of Paonia's Caselle financial management system (Caselle Connect / Clarity, or equivalent modules currently in use) for the period January 1, 2022 through December 31, 2024. To reduce the Town's processing burden and preserve data integrity, I request these records in their native exportable format (CSV or Excel).

1. The Security Audit Log (or equivalently named user activity / security event log), including all user login events, logout events, failed login attempts, permission changes, and administrative actions.

2. The Audit Trail Report (or equivalently named transaction change history / modification log) across all modules, including but not limited to General Ledger, Accounts Payable, Accounts Receivable, Payroll, and Cash Receipts. For each event, I request the fields customarily included in such reports: user, timestamp, module, action (add / modify / delete / void / reverse), transaction or record identifier, and any before/after values the system logs by default.

3. User account records, including account creation dates, modification history, role assignments, permission changes, and deactivation dates for every user account that existed at any point during the request period.

4. Any system reports reflecting deleted, voided, or reversed transactions during the request period.

## Scope clarification

This is a records-management and system-metadata request. I am **not** requesting the underlying financial transactions, journal entry content, bank records, vendor invoices, payroll records, or any protected financial data. Only system activity metadata about when users accessed or modified records, not the content of those records.

Because the requested records are system-generated operational metadata, no accountant-client privilege, deliberative-process privilege, or work-product protection applies. These are not financial records; they are records of who used the financial system and when.

## Format and delivery

Please provide the records as native CSV or Excel exports from Caselle. Printed, scanned, or PDF-rendered versions would unnecessarily increase your processing costs and my inspection cost, and would strip machine-readable fields that Caselle exports natively. If Caselle's standard reports produce these logs in an alternative machine-readable format (XML, JSON, fixed-width text), that is acceptable.

If the Town contends that any portion of the requested records is exempt from disclosure, please identify the specific records withheld and the statutory basis for each withholding, as required by C.R.S. 24-72-204(3).

Please confirm receipt of this request within three business days and provide an estimated cost and completion date pursuant to C.R.S. 24-72-203.
